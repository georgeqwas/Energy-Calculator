# BizNest · Οδηγίες για σένα (runbook)

Τι πρέπει να κάνεις από την πλευρά σου, βήμα-βήμα. Είσαι σε Mac.

---

## Μέρος Α — Να δεις την εφαρμογή τώρα (1 λεπτό)

1. Άνοιξε τον φάκελο `Energy Calculator`.
2. Κάνε **διπλό κλικ** στο `prototype.html` — ανοίγει στον browser.
3. Δοκίμασε: Ρεύμα/Αέριο → πάροχο → kWh → «Υπολόγισε».

> Τα νούμερα τώρα είναι **δείγμα** (βλέπεις το σήμα «SAMPLE» πάνω δεξιά). Γίνονται
> αληθινά μόλις τρέξεις τον scraper (Μέρος Γ).

## Μέρος Γ — Να φέρεις πραγματικά δεδομένα από τη ΡΑΑΕΥ (scraper)

### Γ1. Έλεγξε αν έχεις Python
Άνοιξε **Terminal** (Spotlight → «Terminal») και γράψε:
```
python3 --version
```
Αν δείξει π.χ. `Python 3.12` → ok. Αν λέει «command not found», κατέβασε το Python
από https://www.python.org/downloads/ και ξανακάνε το βήμα.

### Γ2. Πήγαινε στον φάκελο του scraper
```
cd ~/Claude/Projects/"Energy Calculator"/scraper
```

### Γ3. Εγκατέστησε τα απαραίτητα (μία φορά)
```
pip3 install -r requirements.txt
```

### Γ4. Τρέξε τον scraper
```
python3 fetch_tariffs.py
```
- Αν δεις π.χ. `✓ Γράφτηκαν 180 τιμολόγια` → τέλεια. Άνοιξε ξανά το `prototype.html`,
  το σήμα «SAMPLE» φεύγει και βλέπεις αληθινά τιμολόγια.
- Αν δεις `⚠ 0 τιμολόγια` → ο πίνακας φορτώνει με JavaScript. Πήγαινε στο Μέρος Δ.

### Γ5. Επανάληψη κάθε μήνα
Η ΡΑΑΕΥ ενημερώνει τα τιμολόγια **μηνιαίως**. Μία φορά τον μήνα ξανατρέχεις μόνο:
```
cd ~/Claude/Projects/"Energy Calculator"/scraper && python3 fetch_tariffs.py
```
(Αργότερα μπορούμε να το κάνουμε να τρέχει μόνο του — δες README.md → cron / GitHub Actions.)

## Μέρος Δ — Ο πίνακας φορτώνει με JavaScript (η περίπτωσή σου)

Πήρες `ValueError: No tables found`. Αυτό σημαίνει ότι το energycost.gr **δεν στέλνει
έτοιμο τον πίνακα — τον χτίζει με JavaScript** αφού ανοίξει η σελίδα. Ο απλός scraper
βλέπει «άδειο». Δύο λύσεις:

### Δρόμος 1 — Playwright (αυτόματο)

**Τι είναι:** το Playwright οδηγεί έναν **αληθινό browser χωρίς παράθυρο** (headless
Chromium). Ανοίγει τη σελίδα σαν άνθρωπος, περιμένει να φορτώσει ο πίνακας, πατάει
«δείξε όλες τις γραμμές» και τις σελίδες, και δίνει το πλήρες περιεχόμενο στον scraper.

Στο Terminal:
```
cd ~/Claude/Projects/"Energy Calculator"/scraper
pip3 install playwright
python3 -m playwright install chromium
python3 fetch_tariffs.py --browser
```
Αν δείξει «✓ Γράφτηκαν Χ τιμολόγια» → έτοιμο, άνοιξε ξανά το `prototype.html`.

### Δρόμος 2 — Χειροκίνητο dump (σίγουρο, αν αποτύχει το Playwright)

1. Φτιάξε φάκελο `dump/` μέσα στο `scraper/`.
2. Άνοιξε στον **Chrome** καθεμία από τις 4 σελίδες «Καταχωρημένα Τιμολόγια»
   (μενού «Ιστορικό τιμολογίων» στο energycost.gr: Ρεύμα Οικιακό, Ρεύμα Επαγγελματικό,
   Αέριο Οικιακό, Αέριο Επαγγελματικό).
3. Σε κάθε σελίδα, αν υπάρχει επιλογή «Show entries / εγγραφές ανά σελίδα», βάλ’ την στο
   μεγαλύτερο νούμερο (ή «All»). Περίμενε να φορτώσει ο πίνακας.
4. `Cmd+S` → αποθήκευση ως **«Webpage, Complete»** → όνομα ακριβώς:
   - `power_home.html` (Ρεύμα Οικιακό)
   - `power_biz.html` (Ρεύμα Επαγγελματικό)
   - `gas_home.html` (Αέριο Οικιακό)
   - `gas_biz.html` (Αέριο Επαγγελματικό)

   και αποθήκευσέ τα μέσα στον φάκελο `dump/`.
5. Τρέξε:
   ```
   cd ~/Claude/Projects/"Energy Calculator"/scraper
   python3 fetch_tariffs.py --html dump/
   ```

> Όποιον δρόμο κι αν διαλέξεις, στείλε μου τι έγραψε το Terminal (πόσα τιμολόγια ή το
> σφάλμα) και προχωράμε.

## Μέρος Ε — Τι σημαίνει «customer-facing»

Μέχρι τώρα φτιάχνουμε **εσωτερικό εργαλείο** = το χρησιμοποιούν **οι υπάλληλοί σου**
για να εξυπηρετούν πελάτες. Δεν το βλέπει ο πελάτης.

**Customer-facing** = «στραμμένο στον πελάτη». Δηλαδή η ίδια λογική, αλλά φτιαγμένη ώστε
να μπαίνει **ο ίδιος ο πελάτης** και να τη χρησιμοποιεί μόνος του. Αυτό αλλάζει αρκετά:

| | Εσωτερικό (τώρα) | Customer-facing (επόμενο) |
|---|---|---|
| Ποιος το χρησιμοποιεί | υπάλληλοι | οι πελάτες σου |
| Πρόσβαση | τοπικό αρχείο | online σε διεύθυνση (π.χ. app.biznest.gr) |
| Λογαριασμοί | όχι | **login** (ο πελάτης συνδέεται) |
| Γλώσσα/UI | τεχνικό, πλήρες | πιο απλό, με επεξηγήσεις |
| Επιπλέον | σημειώσεις υπαλλήλου | **εκπαίδευση**: τι είναι ρήτρα, χρώματα τιμολογίων, FAQ |
| Θέματα | κανένα | ασφάλεια, προσωπικά δεδομένα (GDPR), hosting, ταχύτητα |

Με δυο λόγια: το customer-facing είναι το δεύτερο στάδιο που είχες πει — όπου ο πελάτης
«συνδέεται, εκπαιδεύεται και βλέπει όλες τις πληροφορίες». Όταν είμαστε έτοιμοι, το χτίζουμε
πάνω στην ίδια βάση δεδομένων/λογική, προσθέτοντας login + απλοποιημένο UI + εκπαιδευτικό υλικό.
